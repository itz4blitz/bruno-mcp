# bruno-truthful-tests

This skill defines the testing philosophy for Bruno collections generated by `bruno-mcp`.

## Rules

- no fake passing
- no weakening expectations to match known product bugs
- no hiding core-flow failures with skips
- passing means the product behavior matches the intended contract
- failing means either the product is wrong or the collection is wrong

## Bug documentation

If a known bug exists:

- keep the assertion on the correct behavior
- add a `known-bug` tag
- document the observed defect in request or collection docs
