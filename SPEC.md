# Bruno MCP Server Spec

## Purpose

`bruno-mcp` provides an MCP server that creates and updates Bruno assets for REST APIs, GraphQL over HTTP, binary uploads, and dependency-aware HTTP suites.

This spec reflects the current implemented and tested scope, not an aspirational future surface.

## Runtime

- Language: TypeScript
- Runtime: Node.js `>=20`
- Transport: MCP over stdio
- Validation: Zod
- Lint/format: `oxlint`, `oxfmt`
- Verification: unit tests, MCP integration tests, Bruno CLI acceptance tests

## Supported Tool Surface

### `create_collection`

Creates a Bruno collection directory with:

- `bruno.json`
- `environments/`
- collection `.gitignore`
- collection `README.md`

### `create_environment`

Creates a Bruno environment file using a plain `vars {}` block.

### `create_request`

Creates HTTP-based `.bru` files with support for:

- methods: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`
- headers
- query params
- auth: `none`, `bearer`, `basic`, `oauth2`, `api-key`, `digest`
- body types: `none`, `json`, `text`, `xml`, `form-data`, `form-urlencoded`, `binary`, `graphql`

### `add_test_script`

Adds or replaces:

- `script:pre-request`
- `script:post-response`
- `tests`

The update path preserves other supported request sections.

### `create_test_suite`

Creates multiple related supported HTTP requests in a suite folder.

Supports explicit dependencies between requests by generating runtime-variable extraction scripts with `bru.setVar(...)` and ordering requests so sources execute before targets.

### `create_crud_requests`

Creates a standard REST CRUD request set for an entity.

### `list_collections`

Recursively scans a root directory for Bruno collections by `bruno.json`.

### `get_collection_stats`

Returns:

- total request count
- request counts by method
- folder list
- environment list

## Supported BRU Surface

### Request blocks

- `meta`
- one HTTP method block
- `auth:*`
- `headers`
- `query`
- `body:json`
- `body:text`
- `body:xml`
- `body:multipart-form`
- `body:form-urlencoded`
- `body:file`
- `body:graphql`
- `body:graphql:vars`
- `script:pre-request`
- `script:post-response`
- `tests`
- `docs`

### Environment blocks

- `vars`

## Explicit Non-Scope

These are not implemented and should not be advertised as supported:

- gRPC
- WebSocket

## Quality Gates

The repo is expected to stay green only when all of the following pass:

1. `npm run format:check`
2. `npm run lint`
3. `npm run typecheck`
4. `npm run test:unit`
5. `npm run test:integration`
6. `npm run build`
7. `npm run test:acceptance`

## Acceptance Standard

The key acceptance bar for request generation is:

- a collection generated by the MCP server must run successfully with `bru run`
- no manual patching of generated `.bru` or environment files is allowed

That behavior is enforced by the acceptance test suite.
